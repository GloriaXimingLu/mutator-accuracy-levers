Mutator lever manuals

An outer-loop mutator (an Opus agent) edits a frozen Harvey legal-agent harness to make it better, scored on a held-out dev gate. Its operating manual is a CLAUDE.md. Below are seven manuals: one per accuracy lever from the deep-research catalog, plus a general all-levers manual. Each retargets the same hard-won protocol at a different accuracy goal — and in every one, accuracy is the objective while cost (and no thrash, no turn-runaway) is the hard gate.

Next: wiring the outer loop into OpenEvolve — tested by replay-simulation

Wiring our mutator’s outer loop into OpenEvolve (the AlphaEvolve-style MAP-Elites + island evolver) forces design choices — above all the feature_dimensions the MAP-Elites grid bins on. Running OpenEvolve with the live outer loop inside it, end-to-end, is far too expensive to ablate. So we test the choices on a replay-simulation instead.

The pool

We already ran several outer-loop variations — each seeded from one lever and evolved into a lineage tree of committed harness variants. Every node has a measured (cost, accuracy), so we know which ones sit on the cost–accuracy frontier.

The trick

Run the real OpenEvolve ProgramDatabase loop — sample a parent, place the child, keep one elite per grid cell, migrate across islands — but a “child” is drawn from the pool (the lineage child of the sampled parent) instead of a fresh LLM rollout. Zero rollouts, zero LLM calls.

The payoff

Now we can A/B the OpenEvolve design knobs — feature_dimensions, feature_bins, island count, exploration/exploitation ratios — in seconds, and check the one thing that matters: does a frontier-best program get surfaced as best_program, or does a bad feature choice bury it?

Simulation mechanics mirror OpenEvolve’s database faithfully: per-island MAP-Elites binning with dynamic min–max feature scaling, one elite per cell decided by a fitness that excludes the feature dimensions, ring-topology migration on a generation interval, and parent sampling split across exploration / archive-exploitation / random. The only substitution is the rollout → a pool lookup.
